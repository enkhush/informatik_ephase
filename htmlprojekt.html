Client-Server-Modell

Das Client-Server-Modell beschreibt eine Möglichkeit, Aufgaben und Dienstleistungen in einem Netzwerk zu verteilen. Programme, die in Clients und Server unterteilt werden, führen diese Aufgaben aus. Der Client kann denn Server nach einer Aufgabe fragen und der Server geht auf die Anfrage ein. Ein Server kann für mehrere Clients gleichzeitig arbeiten.

Ein Server ist ein Programm, das einen Dienst anbietet und der Client ist ein anderes Programm, das diesen Dienst nutzen kann. Die Kommunikation zwischen Client und Server ist abhängig vom Dienst, das bedeutet, der Dienst bestimmt welche Informationen/Daten ausgetauscht werden. Der Client ist aktiv, während der Server passiv ist und auf Anfordungen des Clients wartet, somit ist der Server immer in Bereitschaft.
Die Regeln der Kommunikation eines Dienstes werden durch ein für den spezifischen Dienst angelegtes Protokoll festgelegt.

Das Client-Server-Modell kann so ausgebaut werden das eine Gruppe von Servern (Software), die eine Gruppe von Diensten anbietet.

Funktionsweise

Jede Website hat eine IP-Adresse, die wird nötig um sie im Internet zu adressieren. Das einzige Problem ist, dass wir uns eine IP-Adresse schlecht merken können, da werden Domain-Namen/Computernamen benötigt. Sie sind leichter zu merken und da kommt das DNS (Domain Name System) ins Spiel. Es wird benötigt um von einem Domain Namen zur IP-Adresse zu gelangen. Er liefert uns also, zu einem Domain Namen, die passende IP-Adresse.

Es gibt nicht nur einen DNS Server, sondern Tausende, da das Internet ein dezentrales Netzwerk ist. Das bedeutet, dass jedes Netzwerk ihren eigenen DNS Server besitzt, der eine eigene Liste von Domain Namen und IP-Adressen hat.

Wenn jedes Netzwerk ihren eigenen DNS Server hat, gibt es keine zentrale Datenbank und das wiederum bedeutet, das nicht alle DNS Server einen Domain Namen in eine IP-Adresse auflösen kann. In so einem Fall schickt der DNS Server einem anderen DNS Server eine Anfrage und macht es solange bis ein anderer DNS Server eine "Antwort" zurückschickt.

Funktionsweise

TCP/IP wird in zwei Protokolle aufgeteilt. Dem Transmission Control Protokoll-Suite (TCP) und dem Internet Protokoll-Suite (IP). Die TCP-Suite verarbeitet die Daten von Anwendungen und eine Verbindung zwischen zwei Netzwerkteilnehmern herstellt, während die IP-Suite, die Übertragung und den Empfang von Anwendungsdaten über ein Netzwerk ermöglicht. Die TCP/IP besteht aus einer Kombination von vier Schichten, der Anwendung, dem Transport, dem Internet und dem Internet Zugriff. Einer der Vorteile von TCP/IP besteht darin, dass es Datein jeglicher Größe akzeptiert und in kleiner und handhabbare Brocken teilt, genannt Pakete. Ein Paket besitzt alle nötigen Informationen, für ein Netzwerk, um es an das Ziel zu bringen.

Verwendung

Das TCP ist eines ersten Netzwerkprotokolle und auch das erfolgreichste. Es ist nur mit Grundwissen konfigurierbar, da es universell verwendbar und auf allen Betriebssystem gängig ist. Für die Integration verschiedener Netzwerksegmente* benötigt man bestimmtes Fachwissen und spezielle Hardware (Router).

*Ein Netzwerksegment ist ein Teil eines zusammenhängenden Netzwerkes, das vom Rest des Netzwerkes getrennt ist.

Einordnung in das OSI-Schichtenmodell

Die TCP/IP befindet sich in den Schichten 3 und 4 des OSI-Schichtenmodells.

Die TCP befindet sich in der 4. Schicht des Modells, da es zum Versenden von Datenströmen dient und somit im Bereich des Transportes liegt. Die IP wiederum sorgt dafür, dass das nächste Zwischenziel des empfangenen Pakets ermittelt wird und es auch dorthin weiterzuleiten. Also liegt es in der 3. Schichten, die Vermittlungs/Internetschicht.

OSI-Modell steht für Open System Interconnection Model und definiert den Kommunikationsablauf zwischen zwei "Teilnehmern" in einem Netzwerk. Dieses Referenzmodell besitzt sieben Schichten (Layers), die jeweils klar voneinander abgegrenzte Aufgaben besitzt. Es ermöglicht die Kommunikation zwischen verschiedenen technischen System über unterschiedliche Technologien und Medien. Um dieses Ziel zu erreichen verwendet das Referenzmodell sieben Schichten (Layers) und deren Aufbau ist hierarchisch.

In jeder Schicht werden bestimmte Aufgaben ausgeführt und jede Schicht bietet der über ihr liegenden Schicht Dienste an. Eine Schicht verwendet die Dienste der unter ihr liegenden Schicht und bearbeitet seine eigenen Aufgaben, so wird ein Dienst zur Verfügung gestellt.

Um die Sicherheit, Performance und Zuverlässigkeit einer Kommunikationsverbindung sicher zu stellen werden in den Schichten viele Aufgaben bewältigt. Während zwei Systeme miteinander kommunizieren wird das Modell zweimal oder öfters durchlaufen, weil einmal der Sender und der Empfänger das OSI-Schichten modell berücksichtigen muss.

  • Schicht 1-4: Transportorientiert
  • Schicht 5-7: Anwendungsorientiert
